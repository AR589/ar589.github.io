AR 589 — Week 12

JavaScript: Working with Events

How Events Work

  • User does something, like click a link.
  • Browser reports the event in the form of an event object, supplied to the designer.
  • Designer User can "handle" the event (or not).

What's an Event?

  • A JavaScript Object!
  • As you browse the web, the browser is constantly telling you what the user is doing.
  • You can “listen” for different events.
  • When you the browser tells you about an event, you can “handle” it.

Types of Events

  • Browser Events
  • User Interface Events
  • DOM Events
  • Form Events

See the lecture slides for lists of common events

How to Handle Events

Step 1: Select an element

var myElem = document.querySelector('.myClass');

Step 2: Tell the browser what event you're listening for and tell it what function should handle the event.

myElem.addEventListner('click', myFunction)

Step 3: Write your fucntion!

function myFunction (event) { // Do something here... }

Note: addEventLisener implicitly sends an event object to the function it calls.

The Event Object

  • Gives you a ton of information
  • You will commonly work with event.target and event.type properties
  • Has two methods: event.preventDefault() and event.stopPropagation()

Event Bubbling

An event reaches the "deepest" or most nested element first and then "bubbles" up until it reaches the root document element. You can listen for events on any element the event touches.
